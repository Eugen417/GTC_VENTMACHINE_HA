# GTC Ventmachine для Home Assistant

Пользовательская интеграция для управления приточными установками **Ventmachine** (автоматика GTC, Syberia 5 и аналоги) через протокол Modbus TCP.

## 📥 Установка

> **⚠️ ВНИМАНИЕ:** Перед установкой данной интеграции обязательно удалите или закомментируйте старую YAML-конфигурацию Modbus для этой вентустановки в файлах Home Assistant. Протокол Modbus TCP в контроллерах GTC поддерживает только одно активное соединение. Если старый код продолжит работу, новая интеграция выдаст ошибку `Connection refused`.

[![Open your Home Assistant instance and open a repository maintainer's profile on GitHub.](https://my.home-assistant.io/badges/hacs_repository.svg)](https://my.home-assistant.io/redirect/hacs_repository/?owner=Eugen417&repository=GTC_VENTMACHINE_HA&category=Integration)
1. **HACS**: Нажмите кнопку выше или добавьте репозиторий `https://github.com/Eugen417/GTC_VENTMACHINE_HA` в Custom Repositories.
2. **Настройка**: Добавьте интеграцию через «Устройства и службы», указав IP-адрес контроллера.

## Возможности версии 1.3.1

* **Нативное управление**: Полноценные сущности Climate и Fan для интеграции в стандартные карточки Home Assistant и Apple HomeKit.
* **Семантическая индикация**: Аварии используют встроенные классы Home Assistant (`device_class: problem`), обеспечивая автоматический перевод и правильную раскраску графиков (красный/зеленый).
* **Прозрачность процессов**: Подробное отслеживание каждого этапа работы (открытие заслонок, прогрев, выбег, движение клапанов) с понятными текстовыми статусами.
* **Системная информация**: Автоматическое чтение MAC-адреса, IP и версии прошивки контроллера.
* **Яндекс Алиса**: При помощи компонента [Yandex Smart Home](https://github.com/dext0r/yandex_smart_home) установку легко добавить в Умный дом Яндекса для голосового управления.

## 🌐 Перечень сущностей

### 🎛️ Управление

| Имя в Home Assistant | Описание | Возможности |
| --- | --- | --- |
| **Климат** | Основной термостат | Нагрев / Выкл, выбор температуры (5.0–30.0 °C) |
| **GTC Fan** | Управление вентилятором | Вкл / Выкл, плавная регулировка 1–10 (10–100%) |
| **Уставка температуры** | Прямое задание уставки | 5.0–30.0 °C |
| **Интервал опроса системы** | Настройка частоты опроса Modbus | 2–10 секунд |

### 🌡️ Датчики (Сенсоры)

| Имя в Home Assistant | Описание | Состояния / Ед. изм. |
| --- | --- | --- |
| **Состояние Уст.** | Текущий этап работы системы | Ожидание, Работа, Заслонка и др. |
| **Текущая температура** | Датчик приточного воздуха (T1) | Число (°C) |
| **Целевая температура** | Заданная уставка нагрева | Число (°C) |
| **Наружная температура** | Уличный датчик (T2/T3) | Число (°C) |
| **Температура в помещении** | Датчик в комнате (T_room) | Число (°C) |
| **Влажность** | Относительная влажность | Число (%) |
| **Текущая / Целевая скорость** | Ступень вентилятора | 0–10 |
| **Загрязнение фильтра** | Оставшийся ресурс фильтра | Число (%) |

### ⚙️ Статусы процессов (Текстовые)

| Имя в Home Assistant | Описание | Состояния |
| --- | --- | --- |
| **Статус: Питание** | Общее состояние установки | **Включено / Ожидание** |
| **Заслонки и Клапаны** | Движение механизмов (Воздух, ГВС, ХВС) | **Открывается / Закрывается / Покой** |
| **Этапы запуска** | Предподогрев, Северный старт, Разгон ротора | **Активно / Покой** |
| **Блокировка пульта** | Статус локального управления | **Заблокировано / Свободно** |

### ⚠️ Аварии и ошибки (Бинарные сенсоры)

*Используют класс `Problem`. В истории отображаются как **Проблема (Красный)** / **Норма (Зеленый)**.*

| Группа | Сущности |
| --- | --- |
| **Безопасность** | Пожар, Внешняя авария, Перегрев калорифера / системы, Угроза заморозки (вода/воздух/рекуператор) |
| **Механика** | Вентилятор 1/2, 100% Фильтр 1/2, Нет теплоносителя |
| **Датчики** | Ошибки T1, T2, T3, T4, T5, Датчик предподогрева, Недогрев системы |

## 🤖 Автоматизации (Blueprints)

Для интеграции доступен готовый сценарий (Blueprint), который упрощает создание умных правил управления:

* **Управление по CO2**: Автоматически включает/выключает установку и регулирует скорость вентилятора в зависимости от уровня углекислого газа. Позволяет поддерживать свежесть воздуха и экономить ресурс фильтра, когда в помещении никого нет.

Установить сценарий управления скоростью по CO2 в один клик:
